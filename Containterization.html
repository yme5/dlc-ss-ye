HTTP and HTML

Sehrish Saeed

Feb 28, 2021

HTTP

HTTP and DNS handle the sending and receiving of web files. HTTP is a way that computers communicate with each other. When you open a web browser and you type in the web address, it opens the website you want to visit. When you type in a web address, the computer talks to a server and asks it for a website, and they communicate in HTTP or hypertext transfer protocol. If a user is trying to login to a website, the computer sends a message to the server by the way of a “get” command. Get commands retrieve information from the server to the computer.

HTML

HTML or hypertext markup language is used to tell a computer how to make a page look. It is used to format font, color, alignment, & more. Images and videos are separated from the text and need their URLs copied to be included in the webpage. If there are many images, each request needs an HTTP request which can result in the page opening slowly. Cookies are “invisible” codes that are used to help the computer remember who a user is. Cookie data is essentially and ID card. The browser saves this information and attaches the ID number to the request that is being sent to the server. Since the internet is open, hackers can monitor any private information. By using a secure server, users can prevent snooping and tampering. Digital certificates are ID cards that identify trusted website entities. Browsers will warn users when a website is not secure.
